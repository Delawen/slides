Hi XXX. I want to talk to you about OSGeo: what it is, what we do,...
There are three main categories we can use to clasify geospatial communities: data, leaded by OpenStreetMap; standards, leaded by OGC and software, with OSGeo. They are siblings helping each other to enrich the environment.
Of course, there are other satellite communities around geospatial, like the Linux Foundation. But the undoubtful leader on this area is OSGeo.
OSGeo is huge, composed by lots of groups. It's impossible to describe all of them in 10 minutes.

FOSS4G 2019

It's the year 2020. The CATS have become ZOMBIES, endangering humanity. A group of OSGeo members is safe on a fortified house.

Luckily, their OSGeo knowledge is strong...

So I choosed a use case that covers most of our software projects so you can grasp what our community is. It's the year 2020 and there has been a zombie kitten apocalypse. We are going to follow a group of OSGeo members and how thanks to Free and Open Source they are able to survive.
First thing is to get intel about our surroundings. We need to get food, tools and medicines.

First we need data. Where can we find data? Of course: OpenStreetMap.
It is the biggest open licensed geographic database. The best base we can use for any project.

If we are lucky, we can even have a group of HOTOSM working on our area.
They are a group of volunteers that make sure data in conflictive zones is updated and maintained.

We can use Marble to visualize data from OpenStreetMap. It is a desktop app with routing and search of POIs.

And I know what you are all thinking: this is a zombie apocalypse. The first thing to fail is internet. Sure, but we can keep using guifi.net. It's a free, decentralized and neutral global network project. Anyone can join by setting up an antenna and pointing to an existing node.

So, we can have access to data on our region. Fortunately we have PostGIS to store all this data, the spatial extension for PostGRES. Which, by the way, uses GEOS, a port to C++ of JTS. JTS is an open source Java library that provides a set of fundamental geometric functions.

And we can use GDAL to transform from the OpenStreetMap format to postGIS.

GDAL is a translator library for geospatial data formats

But we have to update the data. There are buildings imploded and many streets are blown up. So we need to gather new to know where are the hordes of kittens and what they are doing.

For example we could get data on where the zombie kittens are hanging.

Now we can analyze this data to extract patterns and see where are our POIs. We can use GRASS and QGIS to run these analysis.
We may conflate and analyze data to extract patterns.
One day we discovered a group of people that didn't survive because they were using limiting licensed software.

And then we discovered another group of survivors, who had an hybrid system that allowed them to survive, but only to some extent. So we left them some OSGeo Live usb to learn and practice, made sure they we have network connectivity to keep talking, and then left.

Now that we are several groups of survivors, we need a system to share data. We deploy a GeoNetwork so they can search our published data and they build a GeoNode with PyCSW to publish datasets that contain analysis over our raw data.

We find more communities and each of them use a different software to share data. One of them specialized in tools, another analyzes the behaviour of the zombie kittens, others keep track of food,...

Some of them use OGC services from Deegree, MapServer or GeoServer and they build their websites over them.

And don't forget pyWPS, to help us share algorithms and analysis to process data with functions defined by other groups. Or simply, using their server power.

They finally defeated the zombie kittens and celebrated victory with a FOSS4G.
