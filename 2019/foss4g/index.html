There are three main categories we can use to clasify geospatial communities: data, mainly represented by OpenStreetMap; standards, leaded by OGC and software, where we are. All three communities are siblings helping each other to enrich the environment.
Of course, there are other satellite communities around geospatial, like the Linux Foundation. But the undoubtful leader on this area is OSGeo.

It's the year 2020. The CATS have become ZOMBIES, endangering humanity. A group of OSGeo members is safe on a fortified house.

Luckily, their OSGeo knowledge is strong...

OSGeo is huge, composed by many initiatives and groups. It's impossible to describe all of them in such a short time. So I choosed a use case that covers at least most of the software projects we have so you can grasp how big our community is. It's the year 2020 and there has been a zombie kitten apocalypse. We are going to follow a group of OSGeo members and how thanks to Free and Open source software they are able to survive.
First thing is to get intel about our surroundings. We need to get food, tools and medicines.

First we need data. Where can we find data? Of course: OpenStreetMap.
It is the biggest open licensed geographic database. The best base we can use for any project.

If we are lucky, we can even have a group of HOTOSM working on our area.
They are a group of mappers that make sure data in conflictive zones is updated and maintained.

We can use Marble to visualize data from OpenStreetMap. Marble is a desktop app with routing and search of POIs. It's a good way to have a global overview of our environment and check which places we want to visit.

And I know what you are all thinking: this is a zombie apocalypse. The first thing to fail is internet. Sure, but we can keep using guifi.net. It's a global network project free, open and neutral. Started on 2004 it covers now many places. Anyone can join by setting up an antenna and pointing to an existing node, for free. As it is decentralized, it is more resilient and can survive catastrophes.

So, one way or another we can have access to data on our region. But how do we process this data? Fortunately we have PostGIS, the spatial extension for PostGRES. Which, by the way, uses GEOS, a port to C++ of JTS. JTS is an open source Java library that provides a set of fundamental geometric functions.

And we can use GDAL to transform from the OpenStreetMap format to postGIS. GDAL is a translator library for raster and vector geospatial data formats

Ahora que ya tenemos los datos, ¿con qué analizamos los datos para detectar los patrones de comportamiento de las hordas de gatitos? ¿Cómo planificamos nuestras misiones de recaptación de productos? Pues tenemos por ejemplo GRASS y QGIS. Ambos nos permiten definir y lanzar algoritmos para procesar todos estos datos y sacar conclusiones de nuestro entorno. ¿Suelen los gatitos moverse de día o de noche? ¿qué tipo de POIs suelen atraerlos?
We may conflate and analyze data to extract patterns.
We can even have specific symbols for the kittens.
One day we discovered a group of people that didn't survive because they were using limiting licensed software.

And then we discovered another group of survivors, who had an hybrid system that allowed them to survive, but only to some extent. So we left them some OSGeo Live usb to learn and practice, made sure they we have network connectivity to keep talking, and then left.

Now that we are several groups of survivors, we need a system to share data. We build a GeoNetwork so they can search our published data and they build a GeoNode with PyCSW to publish datasets that contain analysis over our raw data.

We find more communities and each of them use a different software to share data. One of them specialized in tools, another analyzes the behaviour of the zombie kittens, others keep track of food,...

Some of them use OGC services from Deegree, MapServer or GeoServer and they build their websites over them.

And don't forget pyWPS, to help us share algorithms and analysis to process data with functions defined by other groups. Or simply, using their server power.
